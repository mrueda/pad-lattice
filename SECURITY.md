# Security Policy

## Supported Versions

Pad-Lattice is alpha software. Security fixes are applied to the latest
published alpha and the current `main` branch; older alpha builds are not
maintained.

## Reporting a Vulnerability

Please use [GitHub private vulnerability reporting](https://github.com/mrueda/pad-lattice/security/advisories/new).
Do not publish exploitable details in a normal issue.

Include the affected version or commit, threat scenario, reproduction steps,
impact, and any suggested mitigation. Do not include real credentials, Codex
transcripts, or other private data.

If private reporting is unavailable, open a minimal issue asking the maintainer
to establish a private contact channel, without disclosing the vulnerability.

## Security Model

The maintained threat model, trust boundaries, and deployment guidance are in
the [Security Model](https://mrueda.github.io/pad-lattice/docs/technical-details/security-model)
documentation.
